Detailed Description

Encapsulates the connection to a single player.

This object should have nearly the same lifetime as the socket it represents, except that the object is constructed just before the socket connection is made. A newly-constructed PlayerConnection has no associated player ID, player name, nor host-player status. Once a PlayerConnection is accepted by the server as an actual player in a game, EstablishPlayer() should be called. This establishes the aforementioned properties.
